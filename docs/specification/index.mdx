---
title: Overview
description: Overview of the Fide Context Protocol specification.
---

The Fide Context Protocol defines how context is structured as statements so different systems can build interoperable graph-based world models.

FCP depends on [Fide ID](/id) for identifier rules and on the [Fide Vocabulary](/vocabulary) for entity type codes and entity-specific constraints.

---

This specification defines:

- [Statements](/context-protocol/specification/statements) for the core statement model and statement-level restrictions
- [Attributes](/context-protocol/specification/attributes), [Links](/context-protocol/specification/links), and [Actions](/context-protocol/specification/actions) for the supported statement patterns
- [External Vocabulary](/context-protocol/specification/external-vocabulary) for the vocabulary standards and shorthand conventions used across FCP
